# TSN 拓扑领域语义参考

> **用途**：供 `SKILL.md` 与 Agent 在拓扑阶段把用户的自然语言映射到模板/参数时参照。
>
> **单一事实源声明**：
> - 模板参数 schema、默认值、上下限以 `mcp__tsn_topology__topology_describe_templates` 返回为准，本文件不复述字段定义。
> - 拓扑生成（坐标布局、MAC/IP 派生、链路、四份 artifact、校验）由确定性 MCP（`src-tauri/src/topology_compute.rs`）实现，本文件不定义生成规则。
>
> 本文件只承载 Agent 理解用户意图所需的领域语义，不再作为 builder/validator 的规则来源。

---

## 1. 节点类型与显示名

| `node_type` | 中文名 | 缩写前缀 | 语义说明 |
|---|---|---|---|
| `switch` | 交换机 | `SW` | |
| `networkcard` | 端系统 | `ES` | 用户说的"网卡""端系统""端"都映射为 `networkcard`；"网卡8/网卡9"是新增的端系统节点，不是交换机端口或网卡接口 |
| `server` | 控制器 | `PC` | |

- 首期**排除** `T10` 类型。
- **显示名映射**：画布显示名 = 类型前缀 + `syncName`。`SW-1` 即 `syncName="1"` 的交换机，`ES-4` 即 `syncName="4"` 的端系统。用户提到 `SW-N`/`ES-N` 时按 `syncName` 精确等于 `"N"` 匹配，不要按列表顺序或"第 N 台"折算。

---

## 2. 链路速率参考

`speed` 是整数 Mbps，取值限定 `{10, 100, 1000, 10000}`（标签 10M / 100M / 1000M 千兆 / 10000M 万兆；1000 最常用）。确切取值范围与默认值以 `describe_templates` 的 `dataRateMbps` 参数为准。

---

## 3. 通用分布式拓扑默认互联（场景理解）

当用户说"N 个交换机，每个交换机连接 M 个端系统/网卡/端"或"X 个端系统分配到 N 台交换机"时，除非用户明确说交换机相互独立、不互联，否则默认交换机线型互联：

```text
SW1 -- SW2 -- SW3 -- ... -- SWN
```

这对应 `generic-line` 模板（交换机线型串联 + 每台下挂固定数量端系统）。具体模板选择与参数映射见 `SKILL.md` 的「场景 → 模板选择」决策树。

只有用户明确要求"交换机相互独立""交换机之间不互联""每台交换机单独成星型"时，才省略交换机互联链路（此时由 MCP 模板参数表达，不在本文件定义生成细节）。

## 4. 不涵盖

- 节点配置（GCL / OSS / FWD / MAP / INJECT / INFORM）—— 后续 skill。
- `project.json` 格式 —— 来自外部 xz-nos API，agent 不接管。
- `T10` 节点类型 —— 首期排除。
- Qbu / Qci / CB 等高级特性 —— 远期。
