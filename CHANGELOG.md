# 更新日志

本文件用于在应用内展示客户可见的更新内容。技术名词、文件名和产品名保留原文。

## v0.4.1 - 2026-06-08

### 修复

- 会话列表预览不再显示内部工具调用信息：当某个会话最近一步是工具操作时，列表摘要会显示最近的对话内容，而不是原始的工具调用文本。
- 修复历史会话被误标「迁移失败」：新版本中拓扑直接保存在工程数据库的正常会话，不再被错误判定为「迁移失败」并进入待恢复列表；只有真正损坏的旧版数据才会提示迁移失败。

## v0.4.0 - 2026-06-05

### 新功能

- 会话导出 / 导入界面上线：会话面板新增「导出当前」按钮，可把当前会话保存为独立文件；「导入会话」可打开他人或自己导出的文件，恢复为一个新会话并从原进度继续。导出文件包含完整会话：对话记录（入库时已自动脱敏）、流程进度与拓扑工程。导出后可一键「在 Finder 中显示」找到文件。
- 历史会话迁移失败后可以自助恢复：迁移失败的会话在列表中显示「迁移失败」标记和失败原因，可查看当时的会话数据快照（敏感信息已打码），确认后可重试迁移。
- 智能助手运行期间导出 / 导入入口自动禁用，避免写入中的数据被导出成半成品。

### 修复

- 导入的会话能从原进度继续：此前导入只搬运拓扑、丢失对话与流程进度，导致画布有拓扑却卡在「拓扑草稿」阶段、无法推进到时间同步，打开时还可能因缺对话报错。现在导出携带完整会话，导入后对话、阶段进度与拓扑一并恢复。
- 导出文件内容保证一致性快照：导出瞬间若智能助手正在写入拓扑，文件不会再出现「新旧数据混杂」的情况。
- 导入前对文件做完整校验：超大文件、损坏文件、字段超限或被篡改的文件会被拒绝并给出明确原因，不会污染本机工程数据库。
- 导出临时文件使用唯一名称，不再可能覆盖同目录下用户已有的同名文件。

## v0.3.4 - 2026-06-05

### 修复

- 修正节点指代理解：用户提到 SW-1、ES-4 这类画布上的节点名称时，智能助手现在按名称精确定位对应节点，不再按「第几台」折算导致改错位置（如要求在 SW-1 和 SW-2 之间插交换机却插到了 SW-0 和 SW-1 之间）。

## v0.3.3 - 2026-06-05

### 修复

- 增量拓扑编辑可用了：「在 SW-1 和 SW-2 之间插入一台交换机」这类修改请求，智能助手现在会在已确认的拓扑上做增量调整，不再整表重建导致节点全部重命名。
- 智能助手查询拓扑时能拿到每个节点和链路的完整细节（编号、类型、坐标、连接关系），不再只有数量摘要。
- 拓扑操作格式错误时，智能助手会立刻收到字段级提示和合法操作列表，不再遇到无提示的失败而反复猜测格式。
- 防止误覆盖：新增节点/链路若与已有编号冲突且内容不同会被拒绝并提示正确做法，超时重试同一批操作仍然安全（包括重试时 JSON 格式细微变化的情况）。
- 空操作批次不再触发画布无谓刷新。
- 交互更稳定：选项编号跨轮保持一致、一次聚焦一个决策点、简短确认直接推进，不再出现答非所问的选项重排。

## v0.3.2 - 2026-06-05

### 修复

- 拓扑写入支持安全重试：网络超时后重新执行同一批拓扑操作不再因重复插入而整批失败。
- 阻止坏数据进入工程数据库：链路两端节点必须存在，删除仍被链路引用的节点会被拒绝并提示先删链路。
- 历史会话迁移遇到不完整数据时显式标记失败并进入恢复列表，不再出现「看起来成功但拓扑缺节点」的情况。
- 修复多会话切换与高频更新场景下拓扑画布可能延迟刷新的问题（事件游标并发竞态）。

## v0.3.1 - 2026-06-05

### 其他

- 内部架构清理：移除旧版拓扑计算与导出代码（拓扑能力全部由本机工程数据库服务提供），无功能变化。
- Skill 面板中「流量模板生成」标记为已停用，与流量规划暂时下线的状态一致。
- 新增 CI 防回流检查，防止旧版数据结构重新进入代码库。

## v0.3.0 - 2026-06-04

### 新功能

- 拓扑数据落入本机工程数据库：智能助手生成的拓扑由本机服务直接写入 SQLite，画布、节点详情和链路详情实时读取数据库刷新，不再依赖会话内嵌数据。
- 历史会话自动迁移：升级后首次启动会把既有会话的拓扑迁入工程数据库。
- 新增会话导出 / 导入底层能力（界面入口将在后续版本提供）。

### 修复

- 修复保存会话时拓扑数据可能被清空的问题。
- 修复拓扑阶段指引与拓扑工具版本不一致的问题。
- 提升「流量规划暂时下线」提示条的文字对比度与可读性。

### 其他

- 流量规划与规划导出阶段暂时下线，预计随 Phase B 回归；阶段步骤条以灰态展示。
- Web 浏览器预览模式下智能助手不再可用，请使用桌面版（需要本机工程数据库与拓扑服务）。

## v0.2.1 - 2026-05-26

### 新功能

- 接入外部 TSN 规划服务，支持提交拓扑、流量和规划参数后等待真实规划结果。
- 新增规划任务状态展示，可看到任务运行中、完成、失败、取消等状态。
- 规划结果返回后可生成 planner 输出文件，并补充 INET 仿真所需的 GCL 相关数据。
- 接入真实 TSN 阶段 skill，按拓扑、时间同步、流量规划、模拟仿真分阶段推进。
- 完成桌面端主工作台，支持会话管理、拓扑展示、流量表、导出文件和诊断日志。

### 修复

- 优化规划任务轮询逻辑，临时网络错误后会继续等待结果。
- 修复拓扑和流量规划结果失效后的展示状态，避免旧规划结果误用于新工程。
- 优化拓扑空状态和阶段提示，让首次使用时更容易理解下一步操作。
