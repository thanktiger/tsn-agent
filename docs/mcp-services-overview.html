TSN Agent MCP 服务全景

TSN Agent 当前对外暴露 一个 本地 MCP 服务 —— tsn_topology， 作为 P0 阶段的确定性拓扑领域服务，承担拓扑模板、初始化、校验、artifact 构建、inspect 与 P0 子集 operations。本页汇总服务内容、它在项目里的位置、当前已实现/未实现的能力，以及代码与文档的具体文件位置。

1. 服务定位 — 它在 TSN Agent 中扮演什么

领域服务，而不是 Agent 智能体

tsn_topology 是 无状态确定性领域服务。它不调用大模型、不做自然语言理解、不推进 workflow、不生成完整 TSN project、不导出 HTML，也不处理 time sync/flow/simulation。

它把以前散落在 tsn-topology/ 参考 skill、.claude/skills/tsn-topology 项目内 skill、tsn-stage-runner.ts、topology-factory.ts 的拓扑规则，收敛到一个版本化契约，让 Agent runtime 与客户端共用同一份事实来源。

四阶段工作流里的「拓扑」阶段执行体

TSN Agent 主工作流固定为 拓扑 → 时间同步 → 流量规划 → 模拟仿真。 其中拓扑阶段必须优先经由 tsn_topology MCP 工具产生可信拓扑结果； topology.initialize / topology.apply_operations 在显式授权下回传完整 IntermediateTopology，由 worker 在本地合成 WorkflowStageResult。

其它三个阶段目前仍由本地 stage runner / fake agent 产生结构化结果，尚未拆成独立 MCP domain。

2. 服务内容 — 8 个工具、3 个模板、一个 IntermediateTopology

2.1 MCP 工具清单

SDK fully-qualified 命名是 mcp__tsn_topology__topology_<name>，由 src/topology/topology-service.ts 与 src-node/mcp/topology-tools.ts 保证一致。

工具 允许 full? 用途
topology.describe_templates summary only 返回 P0 模板目录、参数 schema、默认值、约束、场景 tag、示例。
topology.initialize 可 full（授权） 以结构化 templateId + 参数生成 IntermediateTopology；worker 捕获后合成 stage result。
topology.inspect summary only 按稳定 selector 查询节点 / 链路 / 邻接 / 端口占用摘要；命中不唯一时返回 AMBIGUOUS_SELECTOR。
topology.describe_artifacts summary only 返回 4 份 legacy JSON artifact 的数量、大小和计数摘要。
topology.validate_intermediate summary only 校验 IntermediateTopology 的节点 / 链路 / 端口 / server 引用与结构。
topology.build_artifacts summary only 把 IntermediateTopology 构建为 4 份兼容 JSON artifact。
topology.validate_artifacts summary only 校验 legacy JSON artifact 的引用与基本 schema。
topology.apply_operations 可 full（授权 + 去 changeSet） P0 支持 link.delete / node.add / link.add 三类原子 operation，支持 dryRun；其余返回 UNSUPPORTED_OPERATION。

2.2 P0 模板目录

generic-line

通用线型拓扑，多台交换机线型互联，每台挂固定数量端系统。

switchCount 1-12 endSystemsPerSwitch 1-24 10 / 100 / 1k / 10k Mbps

generic-ring

通用环型拓扑，参数与 generic-line 相同，环形互联带基础冗余。

redundant ring

dual-plane-redundant

通用双平面 / 双归属 / 双冗余拓扑。必须显式给出 planes / switches / switchGroups / endSystems / backbone / crossPlaneLinks。

A/B 平面 primary + backup 接入 backbone: line | ring crossPlane: none | paired

⚠️ 旧版 aerospace-redundant 已下线；箭载/航天等场景由 Agent 选 dual-plane-redundant 并展开数量。

2.3 核心数据契约

IntermediateTopology v0

权威拓扑事实模型，schemaVersion = tsn-agent.topology.intermediate.v0。

schemaVersion
固定字符串，工具间共享
metadata
templateId / templateParams / layout / source
nodes[]
id、numericId、name、type（switch/endSystem/server）、ports[]、position、可选 mac/ip
links[]
id、numericId、source/target endpoint、medium、dataRateMbps
diagnostics[]
code / message / severity / path / 可选 details

不含 topologyId / revision；客户端 / project 层自己管理。MCP 完全无状态。

错误 envelope

所有工具失败必返回结构化错误，固定字段：

code message path severity details retryable requiresUserClarification

常见错误码：

UNSUPPORTED_SCHEMA_VERSION INVALID_TEMPLATE_PARAM AMBIGUOUS_SELECTOR UNKNOWN_ENDPOINT_NODE PORT_ALREADY_USED DUPLICATE_NODE_ID DUPLICATE_LINK_ID LIMIT_EXCEEDED UNSUPPORTED_OPERATION FORBIDDEN_RESPONSE_MODE

2.4 输出的 4 份 legacy JSON artifact

topology.json

完整拓扑节点/链路 / 端口 / 坐标

topo_feature.json

拓扑特征摘要（规模、连通性）

data-server.json

数据服务节点描述（兼容/诊断用）

mac-forwarding-table.json

MAC 转发表（兼容/诊断用）

不再生成 mac-forwarding-table.html；也没有 topology.render_mac_table_html 工具。

3. 数据与安全边界 — 哪些不能进模型上下文

Agent-facing 默认 summary

所有工具响应默认只返回 计数 / 摘要 / warning-error summary。

full 只放给两个工具

仅 topology.initialize 和 topology.apply_operations 允许在调用方同时显式设置 responseMode: "full" 且 topologyFullAllowed: true 时返回完整 IntermediateTopology。否则返回 FORBIDDEN_RESPONSE_MODE。

永远不出 worker 的数据

完整 artifact、端口表、MAC 表、完整 changeSet 不进对话、不进诊断日志。topology.apply_operations 的 full 模式还会主动剥离 changeSet。

资源上限（来自 src/topology/limits.ts）

256maxNodes
1024maxLinks
64maxPortsPerNode
32maxOperations
32maxJsonDepth
1 MBmaxIngressPayloadBytes
1 MBmaxArtifactBytes
LIMIT_EXCEEDED超限统一错误码

诊断日志 allowlist

MCP 调用只把 tool 名 / schemaVersion / status / 耗时 bucket / 输入输出计数 / 错误码 / limit 类型 / run id 写进诊断日志。 完整 prompt、凭证、MAC 地址、端口映射、稳定节点/链路 ID、完整 selector、完整 changeSet、大段 artifact 一律不写。

6. 运行与调用

构建与运行（dev / test）

  • npm run build:workeresbuild 把 tsn-topology-server.ts 打包成 src-node/dist/tsn-topology-server.mjs。
  • npm test运行 vitest（含 src-node/mcp/topology-tools.test.ts 与 src/topology/*.test.ts）。
  • npm run cargo:testTauri Rust 测试；运行前必须先 build:worker，否则缺资源。
  • npm run tauri dev桌面 dev 入口；Tauri spawn Node worker，worker 自动发现 MCP server 文件并启用工具。

worker 端发现逻辑

resolveTopologyMcpServerPath(cwd) 按顺序查找：

  1. <cwd>/src-node/tsn-topology-server.mjs
  2. <cwd>/src-node/dist/tsn-topology-server.mjs

找到则注入 mcpServers: { tsn_topology: { command: "node", args: [path] } }， 没找到则 不 注入 MCP，回退到本地 stage runner / domain 调用；不会因为 MCP 缺失阻塞会话。

allowedTools 命名映射

MCP 服务名固定 tsn_topology；工具点号映射成下划线： topology.initialize → mcp__tsn_topology__topology_initialize。 worker 通过 buildAllowedToolsForStage() 仅在 MCP 可用时把这 8 个 fully-qualified 名加入 allowedTools， 并由 topology-tools.ts 里的 assertTopologyToolMapping() 保证映射不漂移。

7. 下一步与决策门

生产打包形态

三选一：① Agent SDK spawn packaged stdio server；② Tauri 管理 sidecar + 私有 IPC / loopback；③ shared package + 本地 adapter。须在 P0 之后定稿。

Node-free 验收

若产品承诺生产环境无系统 Node，claude-agent-worker.mjs 与 tsn-topology-server.mjs 必须同步替换或封装为自包含可执行。

新 MCP domain 入场顺序

time_sync.* / flow.* / simulation.* 必须以独立 domain 需求 + 单独验收门进入；不在 tsn_topology 上做拼接。
