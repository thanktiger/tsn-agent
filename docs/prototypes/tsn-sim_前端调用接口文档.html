    tsn-sim 前端调用接口文档

    通用约定

    项 值
    服务地址 http://100.78.48.43:19080
    接口前缀 /sim
    请求格式 Content-Type: application/json

    错误响应：

    字段 类型 说明
    error.code string 错误码，如 invalid_config、not_found、queue_full、internal_error
    error.message string 错误说明

    示例：

    {
      "error": {
        "code": "invalid_config",
        "message": "task config invalid"
      }
    }

    接口总览

    接口 方法 路径 成功状态码 说明
    探活 GET /sim/healthz 200 查询服务状态
    版本 GET /sim/version 200 查询服务版本
    环境检查 POST /sim/task_check 200 检查仿真和硬件环境
    配置校验 POST /sim/task_validate 200 校验任务配置是否可启动
    启动任务 POST /sim/task_start 202 启动仿真、硬件或二者组合
    查询任务 POST /sim/task_query 200 查询任务状态和结果摘要
    查询指标 POST /sim/task_metrics_query 200 查询 time_sync 曲线和统计
    停止任务 POST /sim/task_stop 202 停止任务

    202 表示请求已受理但任务动作异步完成。task_start 返回 202 后，客户端需要调用 task_query 查询任务最终状态；task_stop 返回 202 后，返回体可能是 stopped，也可能是任务已经完成后的 done/failed/timeout。

    任务请求体

    提交前处理规则：

    • 端口字段 timesync_nodes.master_port、slave_port、port_ptp_enabled 必须是真正的 Array，例如 [1,2]，不接受字符串形式 "[1,2]"。
    • task.type=hardware 或 both 时，当前硬件验证只支持 2 个 switch + 3 个 endSystem；timesync_nodes[].mid 必须与 topology_nodes[].mid 一一匹配。
    字段 类型 必填 说明
    task.task_id string task_start 是；task_validate 否 任务 ID，1-128 字符，匹配 ^[A-Za-z0-9][A-Za-z0-9_.:-]*$
    task.scope string 是 当前固定 time_sync
    task.type string 是 执行目标：simulation、hardware、both
    task.duration number 是 任务持续时间，单位 s；同时用于仿真时长和硬件测试时长
    task.offset_ns_max number 否 最大 offset 阈值，单位 ns；默认 200，建议 1..4095
    topology_nodes[].mid string/number 是 节点 ID；当前同时作为内部 node_id 和硬件 hcp_mid
    topology_nodes[].node_type string 是 节点类型：交换机 switch，端节点 endSystem
    topology_nodes[].mac string/null 否 节点 MAC
    topology_nodes[].ip string/null 否 节点 IP
    topology_nodes[].port_count number/null 是 端口数量
    topology_nodes[].queue_count number/null 是 队列数量
    topology_links[].src_node string/number 是 源节点 ID
    topology_links[].dst_node string/number 是 目的节点 ID
    topology_links[].src_port number/null 是 源端口号，0 <= src_port < port_count
    topology_links[].dst_port number/null 是 目的端口号，0 <= dst_port < port_count
    topology_links[].speed number/null 否 链路速率 Mbps；时钟同步硬件侧不消费，仿真缺失时按渲染层默认值处理
    timesync_domain.gm_mid / timesync_domain[0].gm_mid string/number/null 是 Grandmaster 节点 MID，必须对应某个 topology_nodes[].mid
    timesync_domain.one_step_mode / [0].one_step_mode number/boolean/null 否 0/false 两步法，1/true 一步法
    timesync_domain.fre_switch / [0].fre_switch number/boolean/null 否 频率校正开关
    timesync_nodes[].mid string/number 是 PTP 节点 MID，必须对应某个 topology_nodes[].mid
    timesync_nodes[].master_port[] number[] 否 Master 端口数组；端口号范围 0..31
    timesync_nodes[].slave_port[] number[] 否 Slave 端口数组；端口号范围 0..31，不得与 Master 端口重叠
    timesync_nodes[].port_ptp_enabled[] number[] 否 PTP 使能端口数组；端口号范围 0..31
    timesync_nodes[].sync_period number/null 是 Sync 周期，单位 ms；硬件侧按 OpenSync 2^(-n)s 还原指数，当前整数 ms 可用值为 1000/500/250/125
    timesync_nodes[].measure_period number/null 是 Pdelay 周期，单位 ms；硬件要求为 2^n ms
    timesync_nodes[].report_enable number/boolean/null 否 硬件状态上报开关
    timesync_nodes[].mean_link_delay_thresh number/null 否 平均链路延迟阈值
    timesync_nodes[].offset_threshold number/null 否 节点 offset 阈值，单位 ns；硬件落盘时转 hex

    GET /sim/healthz

    入参：无。

    出参：

    字段 类型 说明
    status string 服务状态：ok、degraded
    queue_depth number 仿真队列长度
    queue_capacity number 仿真队列容量

    GET /sim/version

    入参：无。

    出参：

    字段 类型 说明
    tsn_sim_version string tsn-sim 版本
    api_version string API 版本

    POST /sim/task_check

    入参：

    {}

    出参：

    字段 类型 说明
    status string 总体状态：ok、degraded
    simulation.available boolean 仿真环境是否可用
    simulation.reason string/null 仿真不可用原因
    hardware.available boolean 硬件环境是否可用
    hardware.reason string/null 硬件不可用原因

    POST /sim/task_validate

    入参：同“任务请求体”。task.task_id 可省略；如果传入，也必须满足任务 ID 格式。

    出参：

    字段 类型 说明
    verdict string 校验结论：PASS、WARN、FAIL
    summary string 校验摘要
    task_start_compatible boolean 是否满足 task_start 启动条件
    ready boolean 是否满足本次 type 的启动前条件
    counts object 各严重级别数量
    issues[] array 校验问题列表
    issues[].severity string ERROR、WARN、INFO
    issues[].category string 问题类别
    issues[].code string 问题码
    issues[].message string 问题说明
    issues[].location string/null 问题位置

    POST /sim/task_start

    入参：同“任务请求体”。task.task_id 必填。

    出参：

    字段 类型 说明
    task_id string 任务 ID
    status string 初始任务状态，取值：created、queued、running、done、failed、timeout、stopped
    accepted boolean 是否已受理
    started.simulation.accepted boolean 仿真子任务是否受理
    started.simulation.status string 仿真子任务状态，取值同 status
    started.simulation.job_id string 仿真子任务 ID
    started.hardware.accepted boolean 硬件子任务是否受理
    started.hardware.status string 硬件子任务状态，取值同 status
    started.hardware.job_id string 硬件子任务 ID

    POST /sim/task_query

    入参：

    字段 类型 必填 说明
    task_id string 是 任务 ID，1-128 字符

    请求示例：

    {
      "task_id": "agent-time-sync-001"
    }

    出参：

    字段 类型 说明
    task_id string 任务 ID，1-128 字符
    type string 执行目标，取值：simulation、hardware、both
    status string 任务状态，取值：created、queued、running、done、failed、timeout、stopped
    verdict string 裁决结果，取值：PASS、FAIL、WARN、ERROR
    summary string 任务摘要
    submitted_at string 提交时间
    updated_at string 更新时间
    simulation.status string 仿真子任务状态，取值同 status
    simulation.verdict string 仿真裁决结果，取值同 verdict
    simulation.summary string 仿真摘要
    simulation.error.code string 仿真错误码
    simulation.error.message string 仿真错误说明
    hardware.status string 硬件子任务状态，取值同 status
    hardware.verdict string 硬件裁决结果，取值同 verdict
    hardware.summary string 硬件摘要
    hardware.error.code string 硬件错误码
    hardware.error.message string 硬件错误说明

    POST /sim/task_metrics_query

    入参：

    字段 类型 必填 说明
    task_id string 是 任务 ID，1-128 字符
    source string 否 数据源，默认 both；取值：simulation、hardware、both
    metric string 否 指标类型，默认且当前固定为 time_sync
    mode string 否 查询模式，默认 stats；取值：latest 最新每节点统计、stats 时间段统计、series 分桶曲线、samples 原始样本
    time_axis string 否 时间轴；取值：sim_time、gm_time、relative_time；不传时按数据源选择
    start_time_ns number 否 查询起始时间，非负整数，单位 ns
    end_time_ns number 否 查询结束时间，非负整数，单位 ns；传入时必须 >= start_time_ns
    nodes[] string[]/number[] 否 节点过滤；不传返回全部节点
    bucket string 否 mode=series 的曲线分桶；取值：125ms、1s、5s、10s、1min；不传时服务端按时间范围选择
    only_synced boolean 否 默认 true；硬件样本是否只统计已同步样本，实时曲线通常传 false
    limit number 否 正整数，最大 100000；mode=series 表示每个节点最多返回的 bucket 点数，mode=samples 表示最多返回的原始样本数

    请求示例：

    {
      "task_id": "agent-time-sync-001",
      "source": "hardware",
      "mode": "series",
      "bucket": "1s",
      "only_synced": false
    }

    出参：

    字段 类型 说明
    task_id string 任务 ID
    metric string 固定为 time_sync
    mode string 本次查询模式
    source string 本次查询数据源
    metrics_status string collecting、ready、failed、no_data
    time_axis string 时间轴
    time_range.start_time_ns number/null 查询开始时间
    time_range.end_time_ns number/null 查询结束时间
    runs[] array 数据源运行摘要
    latest[] array mode=latest 时返回的每节点最新统计
    stats[] array mode=stats 时返回的每节点时间段统计
    series[] array mode=series 时返回的曲线数据
    samples[] array mode=samples 时返回的原始样本
    message string 可选说明

    runs[] 字段：

    字段 类型 说明
    source string simulation 或 hardware
    status string collecting、ready、failed、no_data
    sample_count number 样本数
    node_ids[] string[] 节点 ID 列表
    started_at string/null 开始时间
    completed_at string/null 完成时间
    time_axis string 时间轴
    threshold_ns number 裁决阈值，单位 ns

    task_start 配置转换逻辑

    task_start 接收前端提交后的新结构 { task, topology_nodes, topology_links, timesync_domain, timesync_nodes }。服务端先做 schema 校验和任务级校验，再把请求归一化为仿真、硬件各自需要的参数。

    下表只列“前端上传的值被二次处理”的字段。不消费、直接使用、只改属性名、只校验、只补默认值、只落盘，或仿真侧仅直接映射为 INET/NED 属性的字段，均不列入。

    新入参字段 来源 仿真侧转换 硬件侧转换 备注
    task.duration 前端上传，单位 s 转为 sim_time_us = duration * 1_000_000 计算硬件监控截止时间：now + duration 同时作为仿真时长和硬件测试时长
    topology_links[].src_port 前端上传端口号 - 端口号 -> 1 << src_port bitmask 用于生成 TSNLight 拓扑
    topology_links[].dst_port 前端上传端口号 - 端口号 -> 1 << dst_port bitmask 用于生成 TSNLight 拓扑
    topology_nodes[].node_type 前端上传 - switch -> Node_DeviceType=0；endSystem -> Node_DeviceType=3 前端只传这两个值
    timesync_domain.one_step_mode 前端上传 - boolean/number -> "0"/"1" 只接受 0/1 或 boolean
    timesync_domain.fre_switch 前端上传 - boolean/number -> "0"/"1" 只接受 0/1 或 boolean
    timesync_nodes[].master_port 前端上传端口数组 - 端口数组 -> hex bitmask，例如 [0,4] -> "11" bit0 对应端口 0
    timesync_nodes[].slave_port 前端上传端口数组 - 端口数组 -> hex bitmask，例如 [1,3] -> "A" 不得与 master 端口重叠
    timesync_nodes[].port_ptp_enabled 前端上传端口数组 - 端口数组 -> hex bitmask，例如 [0,1,2] -> "7" 为空数组时生成 "0"
    timesync_nodes[].sync_period 前端上传，单位 ms syncInterval = sync_period / 1000 秒 ms -> OpenSync 2^(-n)s 指数，例如 125 -> "3" 当前硬件整数 ms 可用值：1000/500/250/125
    timesync_nodes[].measure_period 前端上传，单位 ms pdelayInterval = measure_period / 1000 秒 ms -> log2(ms) -> hex，例如 2 -> "1"、1024 -> "A" 硬件要求 2^n ms，最大 32768ms
    timesync_nodes[].report_enable 前端上传 - boolean/number -> 硬件字符串 写入 OpenSync 配置
    timesync_nodes[].offset_threshold 前端上传，单位 ns - number -> hex 字符串，例如 1000 -> "3e8" 任务裁决使用 task.offset_ns_max
