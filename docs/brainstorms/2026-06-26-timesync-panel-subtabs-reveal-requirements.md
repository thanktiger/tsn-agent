# 时间同步面板交互重构 — 需求

## 背景与问题

时间同步阶段的右侧配置面板（拓扑画布下方的 config-panel）目前有四处体验断点：

1. **set_gm 后零反馈**：用户/agent 执行 `timesync.set_gm` 后，后端重算落库、`useTimesyncSnapshot` 自动刷新、`treeConfirmed` 翻真，但 UI 不动——面板不展开、不切 tab，用户做完一个动作看不到任何变化、不知道下一步在哪。
2. **软仿/硬仿是两个并排按钮**：硬仿点击只弹 `HARD_SIM_PLACEHOLDER="待接入真实硬件"`，让"未实现"看着像"功能坏了"。
3. **覆盖参数全空白**：`SimOverrideForm` 初始 `form={}`、字段 `value=form.xxx??""`、无 placeholder，用户看不到生效默认（Random/100ppm/60s 在后端兜底），怕"不填=出错"。
4. **硬部署无承载**：没有一个明确的"硬件部署"去处，将来接接口时无锚点。

## 目标

- set_gm 后把用户顺畅引到下一步，但不抢正在别处操作的焦点。
- 用结构（子 tab）讲清"软件仿真 / 硬件部署"是两条**平级**路径，而非两个一样的按钮。
- 让软仿覆盖参数的生效默认值一眼可见、可改、改了可辨。
- 给硬件部署一个诚实有出口的占位，本期不实现其功能。

## 范围

### 本期做（In scope）

- 「时钟同步」tab 改名「时间同步」。
- 时间同步内部分两个**平级**子 tab：软件仿真 / 硬件部署。
- 软件仿真子 tab：承载现有软仿全部内容（覆盖参数表单 + 收敛表 + 抖动曲线 + 解释）。
- 硬件部署子 tab：占位空态 + 回软仿入口。
- set_gm 后的揭示/跳转（分级规则，见 R4）。
- 覆盖参数折叠态 header 生效默认摘要 + 展开预填实值 + 已覆盖标记（R5）。
- 后端暴露软仿默认值供前端读（R6）。

### 不做（Out of scope，本期）

- 硬件部署的真实功能/接口（后续周期）。
- 硬部署空态里的"硬件就绪清单"（SSH 免密 / 远端 INET / 目标机映射）——可作为后续真做硬部署时再填的骨架。
- reveal 抽象成通用事件总线——本期硬编码 set_gm 一个触发点即可（不加假设性扩展点）。
- 覆盖参数的预设方案/分组（当前就 3 个参数，过早抽象）。
- 软仿/硬部署的"软仿收敛才解锁硬部署"gating——**boss 定：平级，有时不需软仿可直接硬部署**。

## 功能需求

- **R1 — tab 改名**：右侧面板「时钟同步」tab 文案改为「时间同步」（含 aria-label 等可达文案）。`activeConfigTab` 的标识键可保持不变，仅改显示名。

- **R2 — 平级子 tab 结构**：时间同步面板内部呈现两个平级子 tab「软件仿真」「硬件部署」，可自由切换，无先后/解锁约束。子 tab 选择是一个新的 UI 状态，归入"随会话切换重置"那一层（见 R9）。容器用 flex 实现，不用 grid（见 R8）。

- **R3 — 软件仿真子 tab 内容**：把现有软仿入口的全部内容（覆盖参数表单、运行态、收敛汇总表、抖动曲线、解释按钮）归到此子 tab。原"软仿"按钮的触发行为不变，只是落位到这个子 tab 内。原"硬仿"并排按钮移除（其语义由"硬件部署"子 tab 承载）。

- **R4 — set_gm 后揭示（分级）**：当处于时间同步阶段、`timesyncSnapshot.domain.gmMid` 由无变有（set_gm 成功，无论用户还是 agent 触发）时：
  - 若 config-panel **收起** → 展开并切到「时间同步」tab、落在「软件仿真」子 tab；
  - 若 config-panel **已开但 activeConfigTab 不是时间同步** → 不切焦点，只在「时间同步」tab 标题挂一个提示标记（脉冲 badge），由用户自己点过去；
  - 若**已开且就在时间同步 tab** → 无额外动作（已经在该看的地方）。
  - badge 在用户首次进入时间同步 tab 时清除。

- **R4a — 揭示与点节点的竞态**：揭示逻辑与既有「点节点强制切 node-props + 展开」是两个独立触发器，按"最近一次动作生效"——set_gm 揭示把 activeConfigTab 设为时间同步；其后用户点节点把它设回 node-props。两者不互相抑制、不并发争抢同一拍。

- **R5 — 覆盖参数默认值可见**：
  - 覆盖参数区默认收起；折叠那一行 header 显示当前生效默认摘要，形如「振荡器 Random · 漂移 100ppm · 时长 60s · 默认」。
  - 展开后表单字段**预填实际默认值**（form 初始化为真实默认而非 `{}`），用户看到的就是会跑的值。
  - 用户改过的字段在 header 摘要里标「已覆盖」（区别于"默认"）。
  - 摘要与预填的默认值都来自 R6 的单一事实源。

- **R6 — 后端暴露软仿默认值**：后端新增一个轻量读取入口，把 `inet_sim_bundle.rs` 的 `DEFAULT_DRIFT_PPM` / `DEFAULT_SIM_TIME_S` / 默认振荡器（`OscillatorKind::Random`）暴露给前端。前端 R5 的摘要与预填读这个源，不在前端另写一份常量（避免后端改默认时前端漂移失同步）。

- **R7 — 硬件部署子 tab 空态**：硬件部署子 tab 显示占位空态，说清三件事：这是什么（硬件部署阶段）、本期不可用、现在请去软件仿真——并带一个直接切回"软件仿真"子 tab 的按钮。不显示"功能坏了"式的裸文案。

- **R10 — offset_threshold 接入（规划期补，boss 授权）**：`offset_threshold` 此前是无单位、无人消费的整数（默认 1000、范围 0..4095）。boss 定其语义为**逐节点收敛偏移阈值（纳秒）**，放宽校验范围。软仿读该列、按各节点阈值判定收敛；收敛表「参考线」与图表阈值带显示实际阈值而非写死 1µs。（编号 R10 接续 R8/R9，本条为规划阶段发现并补入。）

## 关键决策

- **平级而非递进**（boss 定）：软仿/硬部署平级子 tab，不做"软仿收敛才解锁硬部署"——有时不需要软件仿真也可以直接硬件部署。
- **默认值后端出、前端读**（boss 定）：单一事实源在后端，前端通过新读取入口拿默认值渲染摘要与预填。
- **揭示分级而非一律强切**：兼顾 boss 要的"弹出+跳转"与"自动抢 tab 焦点是争议反模式"——按面板当前状态分级。

## 约束

- **R8 — WebKit 布局**：系统 WebKit 下 `grid + overflow` 的 auto 行会 row collapse 塌成 0 高（项目踩过两次）。子 tab 容器与子 tab 内容必须用 flex（`flex-wrap` + `overflow:visible`），切子 tab 用 `align-self:start` + `align-content:start` 防 stretch 跳变。**验证靠 Safari 开 `127.0.0.1:5173` 或真机截图，Playwright 测不出引擎差异。**

- **R9 — state 归属**：沿用既有分层——`configPanelExpanded` / `activeConfigTab` / 新增的子 tab 选择 / `selectedNodeId` 随 `activeSessionId` 变化重置（防 PR#23 的旧会话 id 污染）；`simStatus` + 软仿结果持 App 级（切 tab/切子 tab 不销毁、不取消正在跑的远端命令、切回按状态恢复）；覆盖表单展开态 + 填值是独立 intent，只随会话切换重置为收起 + 默认。

## 成功标准

- set_gm 后（面板原本收起）用户无需手动找面板/切 tab，就落在软件仿真子 tab。
- set_gm 时用户正在节点属性 tab，焦点不被夺走，时间同步 tab 上出现可点提示。
- 不展开覆盖参数也能读到这次软仿会用的三个参数值；展开即见预填默认；改过的项可辨。
- 点进硬件部署子 tab 不会一片空白或像功能坏了，且能一键回软仿。
- 改后端默认值，前端摘要/预填自动跟随，无需改前端。
- dev app 真机（或 Safari 5173）下子 tab 切换无行塌/跳变。

## 依赖 / 假设

- 依赖既有 `useTimesyncSnapshot` 的 `session_db_changed` 刷新链（set_gm 揭示的触发信号来自 `gmMid` 的有无变化）。
- 假设 R6 的后端读取入口是纯读、无副作用（不触发重算）。
- 硬件部署真实功能是后续独立周期，本期空态只需为其留位与引导。

## 待定问题

- 无阻塞性待定项。R7 的"硬件就绪清单"是否在硬部署真做时纳入，留到那个周期再定。
