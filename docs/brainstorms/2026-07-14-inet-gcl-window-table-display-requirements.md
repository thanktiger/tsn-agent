# INET Z3 门控结果按展示定形的数据层 + 门控详情弹窗

## 问题框架

现在 Z3 求解出的门控明细在消费链路上被有损使用：库里 stream_seq 恒写 0（没有流关联）、整个端口的窗口压成一个 durations JSON、读侧只取 ST 门（gate7）行、解析器跳过空 durations 门（丢掉了未调度门的恒态位）。这导致参考竞品的「门控详情」弹窗（四页签：门控可视化 / 流量维度 / 门控表 / 时间参数）在现有数据上画不出来——八队列位图拼不出、窗口和流对不上。

本期把求解结果的存储格式按**展示靶面**重新定形（boss 决策：存储结构以门控详情弹窗需要的内容形式为准），INET 侧解析补齐到这个格式；castup 外部求解器**本期不接**，等数据库格式定稿后由 castup 侧参照我们的格式做返回（对方接口可改）。

## 术语约定

- **窗口**：某端口的门控状态保持不变的一个连续时间段（一条门控明细行）。
- **q0-q7 位图 / 门控状态**：同一含义——该窗口内 8 个队列门各自的开/关，固定 8 位编码。
- **队列**：位图中的某一位（悬浮卡「队列 q3」指该窗口关联流所走的位）。
- **门控操作**：固定常量 `set-gate-states`（802.1Qbv 惯例），展示列，不存库。

## 用户与价值

- 使用者：boss 本人 / 演示对象。规划完门控后打开详情弹窗，能看到每个端口每个窗口的开关、队列位图、这个窗口放行的是哪条流，以及一条流从发出到接收逐跳穿过哪些窗口。
- 现状对照：门控规划子 tab 只有 ST 门的时序图 + 明细表，无位图、无流徽章、无流量维度视图。

## 需求

### 数据层

- **R1 门控明细表（逐窗一行）**：新表存门控明细，每个窗口一行——键 **(session, provider, 节点, 端口, 窗口序号)**（provider 进键，同流集不同 provider 结果共存不碰撞；单 provider 期读查询显式按 provider 过滤）。列含 开始/持续（ns 整数）、q0-q7 位图、关联流列表（JSON，带来源标记）。**配套规划级元数据**（每 (plan, provider) 一份）：cycle_ns（门控周期）、超周期口径、algorithm、生成时间——头部统计的事实源，不靠行聚合反推（尾部空窗缺失时 Σduration 会算短）。表名/列名细节归 plan。
- **R2 解析与读路径补齐**（评审修正后的准确表述）：①解析器**保留空 durations 门**——记录其 initiallyOpen 恒态参与位图合成（现在被跳过，这正是未调度门 q0-q6 恒态位的唯一来源）；②被调度门全量展开成绝对窗口区间（现有解析已覆盖全部 transmissionGate[N]，无需改）；③展示读路径从「flow_plans 按 gate7 过滤」切到新明细表。不改仿真、不改求解。
- **R3 流关联回算**（spike 已验，2026-07-14）：每流发送偏移的数据源**成立**——INET 源码 `GateScheduleConfiguratorBase.cc` 的 `configureApplicationOffsets()` 把求解出的偏移写到 `app.source.initialProductionOffset` 模块参数，`--**.param-recording=true` 会将其记入 .sca（与门参数同一 dump 机制）。回算**按流周期在门控周期内展开全部发送实例（offset + k·period）逐实例匹配窗口**（AE1 的不同周期流场景一个流在超周期内有多组窗口，只匹配首帧会让其余实例误降级）；歧义窗口降级标注「ST 类」级。flow_refs 带来源标记（求解器给出 / 推导 / 类级）。**实现前置**：宿主机服务 plan 通道的 grep 过滤器现在只回门参数行，需扩正则纳入 `initialProductionOffset` 并重新部署服务（services/inet-sim-http/runner.py）；实现期第一步跑真机锁 par 行格式 golden。
- **R4 raw 存档可重放**：每次规划把 .sca 原文落库存 blob。保留策略：**每 (plan, provider) 留最新一份**（不区分 provider 会在 castup 接入后互相顶掉重放源）。体积上界归 plan。
- **R5 新表为唯一事实源（boss 定）**：flow_plans 不再写入 GCL 数据——它过去是验证测试用的过渡形态。全部消费端（展示、软仿验证 pin、agent inspect、删流级联、undo 域）**一次性切到新明细表**；三态语义（ok/solver_failed/no_gating）由规划级元数据承载，不加第四态。verify pin 需要从位图行还原 per-gate 的 initiallyOpen/offset/durations 写回门参数——还原算法归 plan（关键技术决策）。旧表数据不迁移、老工程不做兼容：新表无数据即空态（提示重新规划），软仿验证入口同样按空态禁用。
- **R6 provider 字段**：本期恒填 inet-z3；为 castup 接入留位（已随 R1 进键）。本期不做切换/对比 UI。

### 门控详情弹窗

- **R7 弹窗结构**：门控规划子 tab 保留简版概览 + 新增「门控详情」按钮（全局入口，弹窗打开时筛选器默认「全部」，不携带概览上下文），点开全屏大弹窗，**三页签：门控可视化 / 流量维度 / 门控表**（boss 定：不做时间参数页，多余）。头部元信息：展示周期、超周期、门控端口数、打开窗口数（数据源=R1 规划级元数据 + 行聚合）。顶部「流量」「节点」两个**单选**下拉（默认「全部」）。**筛选语义分页签**：门控可视化/门控表按字面过滤行；流量维度页签**不受节点筛选过滤**（行本身就是流路径节点，过滤会弄断窗口链），节点筛选在该页签退化为高亮定位。
- **R8 门控可视化（甘特）**：行 = 节点.端口，时间轴上画开窗色块，色块带关联流徽章；一个窗口关联多条具体流（非降级）时徽章并排、超过 2 个折叠为「+N」悬浮卡展开——与「ST 类」降级徽章视觉区分。悬浮卡：开关状态、时间区间、持续时间、队列、门控操作（常量）、关联流。
- **R9 流量维度（窗口链）**：按流分组，行 = 该流路径上的每个节点，显示 发/入/出/收 窗口。默认自动选中第一条流（与顶部「流量」下拉同一控件状态）。入/发/收时刻是**纯推导**（上一跳出窗 + 串行化 + 传播 + 处理时延；串行化用 topology_links.speed 算），**不落库**。悬浮卡标注「时间戳为门控窗口边界，不代表报文实际到达时刻」+「数据来源：GCL 规划结果」。**降级态行为（boss 定）**：类级降级的流在本页签**整链隐藏**——流下拉中该流标注「流关联精度不足」且不可选；部分跳降级同样按整链隐藏处理（宁缺毋滥）。
- **R10 门控表（全列）**：列 = 节点/端口/索引/开始(μs)/结束(μs)/持续(μs)/门控操作（常量展示列）/q0-q7 位图圆点/关联流（空窗显示「空窗」）。「导出 Excel」= 前端把当前筛选后的行序列化为 CSV + UTF-8 BOM，无网络请求。真 .xlsx 本期不引入。
- **R11 无时间参数页签（boss 定）**：弹窗只有三页签，不做时间参数页（内容与流量列表详情弹窗重复，多余）。
- **R12 简版概览同源**：门控规划子 tab 现有时序图的数据源切到新表，与弹窗同一份查询口径。
- **R13 弹窗数据三态**：加载中（loading 指示）、查询失败（错误提示 + 重试）、空数据（新表无数据 → 「请重新规划以生成门控明细」，不渲染空图；软仿验证入口同步禁用，口径一致——见 R5）。

- **R15 门控概览卡片组**（2026-07-14 boss 补充，对齐参考截图）：门控规划子 tab 的概览区改为统计卡片组 + 右上按钮组（重新规划 / 门控详情）。八项指标：调度状态（可调度/GCL 已生成）、超周期、业务流数/门控端口数（含涉及队列）、GCL 表项数/打开窗口数、最大门控窗口占用 %、关闭窗口占比 %、最大链路带宽占用 %（流带宽/链路速率逐链路推导取最大）、时延分析（每流端到端时延与裕度、最大值高亮）。时延与带宽由窗口链推导和流表计算得出（同 R9 的诚实边界：规划推导值，非实测）。
  **口径细则（评审补）**：时延分析只覆盖有窗口链的 ST 流，BE/RC 行显示「不适用（无门控窗口）」；类级降级流排除出最大值高亮并提示「N 条流因关联精度不足未计入」（与 R9 宁缺毋滥同调）；max_latency 未填的流裕度显示「未设上限」；推导时延超 max_latency 时**如实显示负裕度**并挂「推导模型与求解器约束口径不同」提示，不遮掩；路径经过 speed 未知链路时带宽卡显示「—（链路速率未知）」。窄面板布局降级（换行/精简）归 plan。

### 流路径指定（2026-07-14 boss 补充并入本期）

- **R16 ST/BE 流显式路径指定**：任何 ST/BE 流可选指定路径（boss 定）——不指定沿现状最短路推导；等长多路径拓扑（现状报 AMBIGUOUS_ROUTE 只能动拓扑消歧）通过指定消歧；非歧义拓扑也可用指定覆盖最短路（如刻意绕路分担负载）。
  - **存储（统一 schema，评审修正）**：flow_streams.paths 列统一为版本化形状 `{"routes":[{node_path, link_seqs},...], "origin":"user"|"system"}`——RC 恒两条（origin=system，保持现状「凭证」语义：验证期照旧重推导复验不相交，**不转事实源**）；ST/BE 显式指定恒一条（origin=user，事实源不重推导）。NULL=系统推导。消费端按形状统一解析，不按 class 分叉。**点名两处既有契约反转**：add_stream 现刻意不透传请求侧 paths（含锁死测试）、UpdateFlowStreamRequest 无 paths 字段——均需推翻，update 校验闸须覆盖 paths 变更。
  - 入口双通道（agent-native 对等，boss 定）：①流详情弹窗加「路径」字段——候选路径下拉（系统枚举可行简单路径，上限与排序归 plan；上限打满时提示「还有未列出路径」）+「系统自动（最短路）」默认项；选项文本固定格式 `节点1→节点2→…`（超长省略号+完整 tooltip）；选中/悬停候选时画布路由高亮联动预览（复用现有高亮出口）。**RC 流该字段只读展示两条冗余路径**并注明「FRER 双路径由算法保证不相交，不可手选」。②agent 的 add_stream/update_stream 工具加可选 path 参数——**同时接受 mid 与唯一 name**（与 talker/listener 口径对齐），无名节点只能用 mid；解析失败逐节点报结构化错误（NODE_NOT_FOUND / NODE_NAME_AMBIGUOUS）。
  - 校验（录入闸）：路径连通、首尾=talker/listener、链路存在、无重复节点；双平面拓扑上路径须整条落在同一平面内（沿现状 ST 锁平面语义）。
  - **消费期复验（评审补）**：全部消费出口解析显式路径时复验 link_seq 存在性 + 端点连续性 + 首尾锚定——拓扑变更后 link_seq 可能悬空甚至被重绑定到另一条链路（initialize 全量重发编号），失效时响亮报「指定路径已失效，请重新指定或改回系统自动」（PATH_STALE 类错误码），**绝不静默回退最短路**；拓扑全量重建（initialize）不级联清指定路径（跨域写破坏撤销一致性），统一走上述惰性失效 + 规划过期标记置位。
  - 全链路生效：规划（Z3 pathFragments）、验证 pin、流关联回算（R3 的匹配路径 = 本条统一解析出口的输出，指定优先）、窗口链推导、路由高亮统一吃「指定优先、缺省推导」的同一路径解析出口（现状 5 处 derive_route 调用簇全部收口）。**验证侧新增机制**：软仿现有 pin 只覆盖双平面首末跳导流，单平面/绕路场景 INET 会自算最短路转发——指定路径的验证需要仿真侧逐跳转发钉死（静态 MAC 转发表注入或等价机制），机制选型归 plan KTD。
  - **改路径/规划字段变更判定收编服务端（评审修正）**：现状「需重新规划」提示是前端弹窗独有 diff，agent 通道触不到——判定移到服务端（add/update/remove 流与改路径时在规划级元数据置 stale 标记并持久化），门控规划子 tab / 门控详情弹窗 / 概览卡片按 stale 标记统一显示过期提示，双入口同源。
  - AMBIGUOUS_ROUTE 报错文案更新：**枚举全部歧义流名** + 引导「请在流详情中指定路径」（可点击深链本期不做）。

## 关键决策（已拍板）

| 决策 | 结论 | 理由 |
|---|---|---|
| 存储格式以什么为准 | 展示靶面（门控详情弹窗，门控表页签列最全） | boss 定；「schema 对不对 = 门控表能否一条查询渲染」是客观验收 |
| castup 怎么接 | 本期不接；格式定稿后 castup 参照库格式做返回 | 对方接口可改，归一成本移到数据源头 |
| 三视图 UI 落点 | 全屏大弹窗，面板留简版+入口 | 信息密度大，对齐参考截图 |
| 流关联精度 | spike 优先确定性回算（多帧实例展开），失败降级类级+来源标注 | 不把推测冒充事实 |
| 推导量存不存 | 入/发/收窗口链现算不落库 | 避免第二事实源；改公式不迁数据 |
| 单位 | 库内 ns 整数（i64），展示层格式化 μs | 浮点相等陷阱；换算收口 |
| 旧数据迁移 | 不迁移，老工程重新规划即可 | 照 dual-plane 库不迁移先例 |
| provider 维度 | 进明细表主键 | 共存不碰撞；对外格式一次定对，避免定稿后破坏性改键 |
| GCL 事实源 | **全部消费端一次性切新表**（展示/verify pin/inspect/删流级联/undo），flow_plans 停写退役 | boss 定；旧表本就是验证测试用的过渡形态，单一事实源避免两表打架 |
| 降级态流量维度 | 降级流整链隐藏 + 「流关联精度不足」不可选 | boss 定；宁缺毋滥，不展示断链 |
| 老工程兼容 | 不做；新表无数据即空态，验证入口同步禁用 | boss 定；重新规划一次即恢复 |
| 时间参数页签 | 不做（弹窗三页签） | boss 定；与流量详情弹窗内容重复，多余 |

## 验收示例

- **AE1**：录 2 条不同周期 ST 流 → 重新规划 → 门控可视化每行是节点.端口、色块带流徽章，**同一流在超周期内的多个发送实例窗口均带徽章**；门控表能看到 q0-q7 圆点位图。注意 INET Z3 场景下「位图全 0 的空窗」预期不出现（非 ST 队列恒开），其空窗形态是「ST 窗之外 q0-q6 开、q7 关」。
- **AE2**（spike 成立前提下）：流量维度页签选 F1：从 talker 的「发」到 listener 的「收」逐跳窗口链完整，悬浮卡带「非实际到达时刻」标注。
- **AE3**：spike 失败或同门多流歧义场景：门控可视化/门控表对应窗口徽章显示「ST 类」，悬浮卡标注来源=类级；流量维度页签中该流整链隐藏、流下拉标注「流关联精度不足」不可选。
- **AE4**：门控表筛选节点后点导出：CSV 行数 = 筛选结果条数，中文列头 Excel 打开不乱码。
- **AE5**（数据层 DoD）：门控表页签的全部列由一条查询（+拓扑显示名 join +规划级元数据）直接渲染，无 per-视图特判。
- **AE6**：老工程未重新规划：弹窗与简版概览均显示「请重新规划以生成门控明细」，不渲染空图；软仿验证按钮同步禁用（不会按旧 flow_plans 数据出结果）。
- **AE7**（R15）：规划成功后门控规划子 tab 显示八张概览卡（调度状态/超周期/流与端口/GCL 表项/窗口占用/关窗占比/链路带宽占用/时延分析），数值与门控详情弹窗同源一致；每流端到端时延与裕度可见，标注为规划推导值。
- **AE8**（R16）：对称拓扑（A→B 两条等长路径）录 ST 流 → 规划报「存在多条等长路径（涉及流：F0），请在流详情中指定路径」→ 详情弹窗路径下拉选择一条（选中时画布高亮预览）→ 重新规划成功；流量维度窗口链与画布路由高亮按指定路径显示。对话里让 agent「把 F1 改走另一条路」同样生效，**且改路后门控规划子 tab 与门控详情弹窗出现同一「需重新规划」过期提示**（stale 标记服务端持久化）。
- **AE9**（R16 消费期复验）：指定路径后删掉路径上一条链路 → 规划/验证/详情展示均响亮报「指定路径已失效」，不静默走最短路；重新指定或改回「系统自动」后恢复。

## 遗留问题

- 传播/处理时延常数取值：**不取 0**（交换机零耗时穿越会让懂行的演示对象质疑全部数据），plan 期取量级合理常数（TSN 交换机常见 1-2μs 量级），并做 sanity check——推导的下一跳「入」时刻应落在该跳开窗内，不满足时悬浮卡加「时延常数与求解器建模不一致」提示。
- 数据规模上界与渲染策略：典型场景窗口数不大（本期演示拓扑 10 端口 × 每端口个位数窗口），门控表暂不做虚拟滚动；plan 期给出上界断言，超界再议。
